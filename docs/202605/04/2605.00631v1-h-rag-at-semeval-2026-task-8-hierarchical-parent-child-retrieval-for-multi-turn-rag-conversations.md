## 摘要
我们展示了 H-RAG，这是我们提交至 SemEval-2026 第 8 题（MTRAGEval）的系统，涵盖了任务 A（检索）与任务 C（基于检索段落的生成）。任务 A 评估独立的检索质量，而任务 C 则评估在多轮对话环境中的端到端检索增强生成（RAG），要求生成准确的回答并忠实地基于检索到的证据。我们的方法实现了一个分层的父子级 RAG 流水线，在生成过程中将细粒度的子级检索与父级上下文重建分离。文档被分割为句子级重叠的子块（child chunks），同时保留完整文档作为父级单元，以提供连贯的上下文。检索过程结合了稠密-稀疏混合搜索、可调权重以及基于嵌入的相似度重评分机制，用于对子块进行检索。检索到的证据在父级进行聚合，并输入至经指令微调的语言模型以生成回答。H-RAG 在任务 A 上取得了 nDCG@5 分数 0.4271，在任务 C 上取得了调和平均分 0.3241（RB_agg: 0.2488, RL_F: 0.2703, RB_llm: 0.6508），表明检索配置与父级聚合在多轮 RAG 性能中具有重要作用。

## Abstract
We present H-RAG, our submission to SemEval-2026 Task 8 (MTRAGEval), addressing both Task A (Retrieval) and Task C (Generation with Retrieved Passages). Task A evaluates standalone retrieval quality, while Task C assesses end-to-end retrieval-augmented generation (RAG) in multi-turn conversational settings, requiring both accurate answer generation and faithful grounding in retrieved evidence. Our approach implements a hierarchical parent-child RAG pipeline that separates fine-grained child-level retrieval from parent-level context reconstruction during generation. Documents are segmented into overlapping sentence-based child chunks, while full documents are preserved as parent units to provide coherent context. Retrieval combines hybrid dense-sparse search, tunable weighting, and embedding-based similarity rescoring over child chunks. Retrieved evidence is aggregated at the parent level and supplied to an instruction-tuned language model for response generation. H-RAG achieves an nDCG@5 score of 0.4271 on Task A and a harmonic mean score of 0.3241 on Task C (RB_agg: 0.2488, RL_F: 0.2703, RB_llm: 0.6508), underscoring the importance of retrieval configuration and parent-level aggregation in multi-turn RAG performance.

## 一、研究核心问题与整体背景

- **研究动机**  
  检索增强生成（Retrieval-Augmented Generation，RAG）是解决大型语言模型（LLM）幻觉及知识滞后的关键技术，通过外部文档检索提高事实性与可追溯性。  
  然而，在 **多轮对话场景** 中，RAG 受到以下挑战：  
  1. 用户问题依赖上文、存在指代或省略；  
  2. 意图随对话演化，导致检索难度上升；  
  3. 检索粒度与生成上下文之间存在矛盾：  
     - 细粒度（短片段）检索精准，但上下文不足；  
     - 粗粒度（整文档）检索连贯，但匹配粗糙。  

- **研究目标**  
  SemEval-2026 Task 8（MTRAGEval）提出多轮 RAG 的评测框架，分为：
  - **Task A**：纯检索质量评估；
  - **Task C**：端到端（检索＋生成）评估。  
  本文的目标是提出一种兼顾检索精度和上下文连贯性的分层检索方案，以提升多轮 RAG 的整体性能。

## 二、方法论：分层父子级（Hierarchical Parent–Child）RAG 流水线

### 1. 核心思想
- 将文档拆分为 **子级（child）子块** 供检索使用；
- 保留原始 **父级（parent）文档** 以支持生成阶段的上下文重建；
- 检索在子级进行，生成在父级进行，从而平衡精度与上下文完整性。

### 2. 方法结构
- **阶段 1：文档分层预处理（Ingestion）**
  - 使用规则分句，将文档分割为 **三句一组、步幅为两句** 的重叠子块；
  - 每个子块嵌入为向量（embedding）并存入向量库；
  - 父文档与子块通过元数据关联。

- **阶段 2：多轮查询检索**
  - 由 API 管理器调用历史对话上下文；
  - 对含指代的提问通过轻量 LLM（GPT-5, 温度 0.2）重写为独立查询；
  - 执行 **混合稠密–稀疏检索**（dense + sparse）：
    - 权重参数 α = 0.7；
    - 稠密检索采用 BAAI/bge-large-en-v1.5 向量；
  - 对候选子块进行 **基于嵌入相似度的重排序（rescoring）**；
  - 初选候选数量 k = 50。

- **阶段 3：父级聚合与排序**
  - 将子块对应回父文档，父文档得分为其最高子块得分；
  - 两种排名策略：
    1. 先按子块排名、后聚合；
    2. 先聚合父文档、再对父文档再评分；
  - 选取 top-n = 5 父文档用于生成上下文。

- **阶段 4：生成与回答**
  - 使用 GPT-5（温度 0.7, max tokens = 4096）生成对话式回答；
  - 避免引用标记或提及“上下文”，在信息不足时自然拒答；
  - 不进行任何微调，仅使用零样本推理。

## 六、主要结论与发现

  - **生成流畅度高**（RB_llm最高），

- **关键发现**
  1. 检索配置与父级聚合对整体性能影响最显著；
  2. 多轮 RAG 的瓶颈在于**证据选择与生成对齐**；
  3. 语义流畅 ≠ 事实忠实，retrieval-grounding gap 依然存在；
  4. 适当的混合权重与父级再评分可显著提高召回与 nDCG 指标。
